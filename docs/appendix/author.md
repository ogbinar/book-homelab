# About the Author

## Myk — Tinkerer, Experimenter, Recycler

**Myk is not a hardcore sysadmin or enterprise architect.** He's just someone who likes to tinker, experiment, and see how far existing computers can still go before throwing them away.

---

## The Journey

What started as curiosity slowly turned into:

- A **homelab** running on recycled hardware
- A **personal cloud** built from open-source software
- A **small AI lab** constructed through trial and error

The common thread? **Nothing was bought new unless absolutely necessary.** Old laptops, second-hand mini PCs, recycled SSDs — that's the backbone of this entire guide.

---

## The Philosophy

> **"The best server is the one you already own."**

This guide is built on a few core beliefs:

### 1. Hardware Doesn't Matter (As Much As You Think)
You don't need a ₱50,000 NUC or a dual-Xeon rig. You need curiosity and 30 minutes a week. Myk's entire homelab started with a ₱0 old laptop from high school.

### 2. Open Source Is Freedom
Vendor lock-in is the enemy. If you can't inspect the code, if you can't host it yourself, if you're at the mercy of a Terms of Service update — you're not in control.

### 3. Breaking Things Is Learning
Every error message is a lesson. Every failed deployment is a story. Every "it works!" moment is a victory. Myk has broken more servers than he can count. That's the point.

### 4. Community Over Competition
This guide wouldn't exist without the spirit of **Data Engineering Pilipinas (DEP)** — a community built on:
- Learning openly
- Sharing knowledge freely
- Helping people grow **without gatekeeping**

---

## The Inspiration

This guide is heavily inspired by the spirit of **[Data Engineering Pilipinas (DEP)](https://dataengineering.ph/)**.

DEP proved that:
- Filipino tech communities can thrive
- Knowledge sharing doesn't require gatekeeping
- Learning openly benefits everyone

> **The goal is not perfection.** The goal is to build, learn, break things, fix them, and hopefully help others do the same.

## What Myk Uses

| Category | Tool | Why |
|----------|------|-----|
| **Server OS** | Ubuntu Server 24.04 LTS | Stability, documentation, industry relevance |
| **Container Runtime** | Docker + Docker Compose | Simplicity, portability, ecosystem |
| **Reverse Proxy** | Caddy | Auto-SSL, simple config, no YAML hell |
| **Monitoring** | Prometheus + Grafana | Industry standard, powerful, free |
| **Backup** | rsync + Restic | Simple + encrypted, local + cloud |
| **Hardware** | Second-hand mini PCs, recycled laptops | Cost-effective, sustainable, proves the point |
