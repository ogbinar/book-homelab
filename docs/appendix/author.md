---

## The Philosophy

> **"The best system is the one you can understand, maintain, and keep running."**

This guide is built on a few core beliefs:

### 1. Practicality Beats Prestige
You do not need expensive hardware to start. You need a clear purpose, a budget that makes sense, and enough discipline to keep the setup maintainable.

### 2. Open Source Is Leverage
If you can inspect it, host it, and modify it, you are in a better position to learn from it and own the result.

### 3. Small Experiments Build Real Skill
Each failed deployment, broken config, or weird edge case becomes part of the learning loop. The point is not to look perfect. The point is to get better.

### 4. Community Makes the Work Better
This guide would not exist without the spirit of **Data Engineering Pilipinas (DEP)** — a community built on:
- Learning openly
- Sharing knowledge freely
- Helping people grow **without gatekeeping**

---

## The Inspiration

This guide is heavily inspired by the spirit of **[Data Engineering Pilipinas (DEP)](https://dataengineering.ph/)**.

DEP proved that:
- Filipino tech communities can thrive
- Knowledge sharing does not require gatekeeping
- Learning openly benefits everyone

> **The goal is not perfection.** The goal is to build, learn, break things, fix them, and hopefully help others do the same.

## What Myk Uses

| Category | Tool | Why |
|----------|------|-----|
| **Server OS** | Ubuntu Server 24.04 LTS | Stability, documentation, industry relevance |
| **Container Runtime** | Docker + Docker Compose | Simplicity, portability, ecosystem |
| **Reverse Proxy** | Caddy | Auto-SSL, simple config, no YAML hell |
| **Monitoring** | Prometheus + Grafana | Industry standard, powerful, free |
| **Backup** | rsync + Restic | Simple + encrypted, local + cloud |
| **Hardware** | Second-hand mini PCs, recycled laptops | Cost-effective, sustainable, and easy to justify |

## The Real Story

This guide exists because:

1. **Someone needed a practical way to learn** — without making the process feel abstract or academic
2. **Someone needed a budget-aware path** — because the cheapest sustainable setup is usually the one that gets used
3. **Someone needed to own their data** — because SaaS is useful, but it should not be the only option
4. **Someone needed to build real skills** — and homelabs are still one of the fastest ways to do that

That someone is Myk. But the lessons are for anyone who wants to build something real.
